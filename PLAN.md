# 调解服务网站 · 引流与上线全套方案

> 目标：**通过网站引流，获取调解服务客户线索（lead），并促成成交。**
> 本文档覆盖：策略定位 → 网站结构 → 设计规范 → 技术实现 → SEO 引流 → 转化 → 上线部署 → 内容运营 → 合规 → 时间线与预算。

---

## 0. 核心思路（先看这一段）

引流站不是"公司介绍页"，而是一台**获客机器**。它只做两件事：

1. **让有纠纷、在搜索"怎么解决"的人找到你**（流量 = SEO + 内容 + 投放）。
2. **让他们留下联系方式或直接咨询**（转化 = 信任 + 清晰 CTA + 低门槛表单）。

衡量成功的唯一指标：**每月获得多少条有效咨询线索 / 每条线索的成本**。所有设计决策都服务于这个指标。

---

## 1. 策略定位

| 项目 | 建议 | 说明 |
|---|---|---|
| 服务范围 | 综合调解（家事 / 商业 / 社区民事） | 首页分流，各设独立服务页便于 SEO |
| 目标客户 | 正在经历纠纷、想"低成本、快、不撕破脸"解决问题的人 | 调解 vs 诉讼的核心卖点 |
| 核心卖点 | 中立 · 保密 · 比打官司快且便宜 · 维系关系 | 反复强化 |
| 品牌名（占位） | 和合调解中心 | 替换为你的真实品牌 |
| 一句话定位 | "不打官司，也能把事说清、把钱要回、把关系保住。" | 用在首页 hero |

**差异化关键**：潜在客户大多分不清"调解 / 仲裁 / 诉讼/律师"。谁能把这件事讲明白、降低他们的决策焦虑，谁就赢得线索。

---

## 2. 网站结构（信息架构）

```
首页 index.html              ← 主引流落地页，承接所有流量
├── 服务 services.html        ← 三大服务总览 + 各自锚点
│   ├── 家事调解
│   ├── 商业/合同调解
│   └── 社区/民事调解
├── 调解流程 process（首页锚点 + 详述）
├── 关于我们 about.html        ← 调解员资质、信任背书
├── 收费说明 pricing（锚点）   ← 透明定价降低咨询门槛
├── 常见问题 faq（锚点）       ← 同时是 SEO 富矿
├── 知识中心 /articles/        ← 内容引流核心（每篇=一个搜索入口）
│   ├── 离婚调解和打离婚官司有什么区别？
│   ├── 合同纠纷找调解还是起诉？一文讲清
│   └── 调解协议有法律效力吗？
└── 联系/预约咨询 contact      ← 转化终点（表单 + 电话 + 微信）
```

**每个页面都必须有 CTA**（"免费咨询""预约调解"），不让用户走到死胡同。

## 3. 设计规范

### 视觉基调
调解 = **信任、专业、平和、中立**。避免冷硬的"律所黑金"，也避免廉价感。

| 元素 | 规范 |
|---|---|
| 主色 | 沉稳蓝绿 `#1f6f6b`（信任+平和），辅以暖中性色 |
| 强调色 | 暖橙/赭 `#d98a4e`（用于 CTA 按钮，制造点击欲） |
| 中性色 | 文字 `#2b2f33`，背景 `#fbfaf7`（米白，比纯白更柔和） |
| 字体 | 中文：思源黑体/系统无衬线；标题可用稍重字重 |
| 圆角 | 12–16px，柔和不尖锐 |
| 图片 | 真实人物握手/沟通/调解场景，避免廉价图库感 |
| 留白 | 充足，传递"从容、专业" |

### 关键页面区块（首页，从上到下）
1. **Hero**：一句话定位 + 主 CTA（免费咨询）+ 信任标签（保密/中立/资质）
2. **痛点共鸣**："你是否正在经历……"（拉近距离）
3. **为什么选调解**：调解 vs 诉讼对比表（时间/费用/关系/隐私）
4. **三大服务**：卡片式，各带 CTA
5. **调解流程**：4–5 步可视化，消除未知焦虑
6. **信任背书**：调解员资质、成功案例数、客户评价
7. **收费说明**：透明，降低咨询门槛
8. **FAQ**：解答顾虑 + SEO
9. **最终 CTA + 咨询表单**：姓名/电话/纠纷类型/简述
10. **页脚**：联系方式、备案号、隐私声明

### 响应式
移动端优先（多数流量来自手机搜索）。CTA 在移动端固定底部常驻。

## 4. 技术实现

**选型：静态网站（HTML/CSS/原生 JS）**——理由：加载快（SEO 排名因素）、零运维、可部署到任意免费/低价主机、易迁移。

```
website/
├── index.html
├── services.html
├── about.html
├── articles/
│   ├── index.html
│   └── *.html
├── css/styles.css
├── js/main.js
├── assets/img/
├── robots.txt
├── sitemap.xml
└── PLAN.md
```

后续若需在线预约排期/支付，再升级到 Next.js + 表单后端（见第 10 节"演进路线"）。

**表单后端方案**（静态站也能收线索）：
- 最快：Formspree / Tally / 腾讯问卷，零代码接收咨询。
- 国内：可接企业微信/微信客服 + 表单服务。
- 进阶：自建 Serverless 函数（Vercel/云函数）写入飞书多维表格或数据库。

---

## 5. SEO 引流方案（自然流量，长期主力）

1. **关键词布局**：围绕"问题型搜索"而非"品牌词"。例：
   - "离婚不想打官司怎么办"、"合同纠纷调解流程"、"调解协议有没有法律效力"、"欠款不还又不想起诉"。
2. **每个关键词 = 一篇知识文章**（/articles/）。这是免费流量的核心引擎，持续产出。
3. **技术 SEO**：
   - 每页独立 `<title>` 与 `meta description`。
   - 结构化数据 `LegalService` / `FAQPage` JSON-LD（已内置）。
   - `sitemap.xml` + `robots.txt`（已内置），提交到百度/Google 站长平台。
   - 语义化 HTML、图片 alt、移动友好、秒开。
4. **本地 SEO**：注册百度地图/高德/Google Business，填写"XX市调解"，吸引本地客户。

## 5b. 付费 / 社交引流（快速起量）

- **搜索竞价**：百度/Google Ads 投"问题型"关键词，落地到对应服务页。
- **内容平台**：知乎、小红书、抖音/视频号发"调解科普 + 案例"，主页挂网站。
- **本地与转介**：律所、社区、HR、物业等转介渠道，网站作为承接。

---

## 6. 转化优化（把流量变线索）

- **CTA 永远可见**：顶部、每屏一个、移动端底部常驻。
- **低门槛表单**：只要"电话 + 纠纷类型 + 一句话描述"，越短转化越高。
- **多通道**：表单 + 电话 + 微信二维码，满足不同习惯。
- **信任前置**：资质、案例数、评价、保密承诺紧贴 CTA。
- **即时响应承诺**："1 个工作日内回电"。
- 后续埋点（见第 8 节）追踪每个 CTA 的转化率，持续优化。

## 7. 上线部署全流程（Step by Step）

### 步骤 1 · 域名
- 注册域名（阿里云/腾讯云/Namecheap）。建议含"调解/mediation/品牌"易记词。
- 国内主机需**域名实名认证**。

### 步骤 2 · 选主机（按场景三选一）
| 方案 | 适合 | 成本 | 备案 |
|---|---|---|---|
| **国内云 OSS/虚机**（阿里/腾讯） | 主攻中国大陆、要百度排名 | 低 | **必须 ICP 备案**（约 2–3 周） |
| **Vercel / Netlify / Cloudflare Pages** | 海外或不强依赖大陆访问 | 免费起步 | 免备案 |
| **GitHub Pages** | 个人/测试 | 免费 | 免备案 |

> ⚠️ 面向中国大陆且用国内主机：**必须先 ICP 备案**，否则无法开通访问。提前办，别卡上线。

### 步骤 3 · 部署
- 静态站：把文件上传到主机 / 连接 Git 自动部署（Vercel 等推送即上线）。
- 配置 **HTTPS**（Let's Encrypt / 主机自带），全站强制 https。

### 步骤 4 · 接表单后端
- 注册 Formspree/Tally，把表单 action 指向其接口，测试能否收到线索（含手机通知）。

### 步骤 5 · 接统计与站长
- 部署 **统计**（百度统计 / Google Analytics 4 / Plausible）。
- 提交 **sitemap** 到百度站长 + Google Search Console，请求收录。
- 设置**转化目标**（表单提交 = 转化），用于衡量 ROI。

### 步骤 6 · 上线前检查清单
- [ ] 所有链接可点、无死链
- [ ] 移动端显示正常
- [ ] 表单能真实收到测试线索
- [ ] 电话/微信正确可联系
- [ ] title/description 每页唯一
- [ ] HTTPS 正常、无混合内容警告
- [ ] 统计代码已生效
- [ ] 备案号/隐私声明已放页脚（国内）

### 步骤 7 · 上线后
- 提交收录、铺第一批文章、开第一轮小预算投放测水温。
- 每周看：访问量、来源、表单提交数、咨询成交数。据此迭代。

---

## 8. 数据与运营

- **每周指标**：UV、流量来源、跳出率、表单提交数、有效线索数、成交数、单线索成本。
- **内容节奏**：每周 1–2 篇知识文章，半年内累积 30–50 篇形成自然流量护城河。
- **A/B 优化**：标题、Hero 文案、CTA 颜色/文案、表单字段数。

---

## 9. 合规与风险（重要）

- 调解/法律相关服务多有**广告与执业合规**要求：不夸大承诺结果（避免"100%胜""保证要回欠款"等绝对化用语）。
- 标注调解的性质与边界（非诉讼、非法律意见，必要时建议咨询律师）。
- **隐私合规**：表单收集个人信息须有隐私声明、合法存储、最小收集。
- 国内：ICP 备案、可能的经营性备案；用语遵守《广告法》。
- 客户信息**严格保密**——这既是合规也是核心卖点。

## 10. 时间线与演进路线

| 阶段 | 时间 | 产出 |
|---|---|---|
| **第 1 周** | 建站 | 落地页 + 服务页 + 3 篇文章 + 表单（本仓库已开始） |
| 第 2 周 | 上线 | 域名/主机/备案启动、部署、统计、收录 |
| 第 3–4 周 | 起量 | 铺内容、开小额投放、本地 SEO |
| 1–3 月 | 优化 | 按数据迭代文案/CTA，扩内容库 |
| 3 月+ | 演进 | 在线预约排期、案例库、客户后台（升级 Next.js + 后端） |
